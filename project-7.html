Abstract
Research Summary

The rapid rise of electric vehicles (EVs) underscores the need for reliable systems to monitor critical components like electric motors. This thesis proposes a novel predictive health monitoring system integrating the computational power of the Raspberry Pi 4, the versatility of the ESP32 microcontroller, and a multi-sensor array — collecting real-time data on motor temperature, vibration, RPM, current and voltage. Machine learning algorithms including Random Forest, LSTM networks, and CNNs analyse this data to detect anomalies, classify motor conditions, and predict maintenance needs. The system is validated through real-world sensor data collection and Grafana visualisation, contributing to cost-effective predictive maintenance for sustainable EV technology.

System Design

Every layer engineered for real-time fault awareness — from the sensor bus to the ML inference pipeline.

Raspberry Pi 4 (4GB)

Central processing unit running Raspbian OS with Python-based ML inference, Flask web UI, and real-time data pipeline management.

Handles all low-level sensor interfacing over I2C/SPI/ADC. Dual-core 240MHz with on-board Wi-Fi for HTTP push to InfluxDB every 3 seconds.

DS18B20 Temperature

1-Wire digital temperature sensor with ±0.5°C precision, detecting motor overheating conditions in real time.

1-Wire · ±0.5°C · Waterproof

MPU-6050 IMU

6-axis accelerometer + gyroscope for vibration and orientation monitoring. Identifies mechanical faults like bearing wear or motor misalignment via I2C.

ACS712 Current & Voltage

Hall-effect current sensor (30A range) alongside a voltage divider circuit. Tracks electrical anomalies — overcurrent, voltage sag and short-circuit events.

ADC · RMS · Hall-Effect
🔄

NK200 / Hall RPM Sensor

Optical pulse counter with interrupt-driven RPM calculation at ~20Hz resolution. Hall effect backup for validation.

20 pulses/rev · ISR · 50ms

High write-throughput time-series database on Raspberry Pi. Stores voltage, current, RPM, temperature and gyro streams with nanosecond timestamps.

Real-time gauge charts and time-series graphs for all motor parameters. Threshold-based alerts for fault conditions. Directly queries InfluxDB.

Gauge · Time-series · Alerts

Machine Learning

Fault Classification Models

Three complementary algorithms trained on the collected motor dataset to cover temporal, spatial and pattern-based fault signatures.

01

Random Forest

Ensemble classifier for tabular sensor readings. Robust to noisy ADC data, handles feature importance ranking across current, temperature and RPM channels.

02

LSTM Network

Recurrent neural network targeting sequential temporal patterns in motor data. Ideal for catching gradual degradation and early-stage fault trends over time windows.

03

1D-CNN

Convolutional neural network applied to vibration time-series. Extracts local spatial patterns from MPU-6050 gyro streams to detect bearing wear and imbalance.

Project Status

Development Timeline

From thesis proposal through hardware assembly to data collection and ML integration.

PHASE 01 · DONE

Thesis Proposal & Literature Review

Reviewed 8+ published studies on SVM, LSTM and CNN-based EV motor fault detection. Identified research gap in affordable embedded-system solutions. Proposal approved by Supervisor.

PHASE 02 · DONE

Hardware Assembly & Wiring

Assembled ESP32 with DS18B20, MPU-6050, ACS712 current sensor, NK200 RPM sensor and OLED display. Wired 12V → 5V buck converter. Verified all sensor readings via Serial monitor.

PHASE 03 · DONE

Firmware Development (ESP32)

FreeRTOS-based firmware with interrupt-driven RPM counter, I2C polling for MPU-6050, RMS current computation, 5-second gyro/current calibration on boot, and InfluxDB HTTP push every 3s.

PHASE 04 · DONE

Data Pipeline: InfluxDB + Grafana

Deployed InfluxDB on Raspberry Pi 4. Validated voltage, current and RPM data push from ESP32. Built Grafana dashboards with gauge charts and time-series graphs for all 8 sensor fields.

PHASE 05 · DONE

Dataset Collection & Correlation Analysis

Collected real EV motor dataset (MY_EV_DATASET.csv). Plotted Temperature vs RPM, Motor Speed vs Voltage, and correlation heatmap. Confirmed meaningful sensor relationships for ML training.

PHASE 06 · IN PROGRESS

ML Model Training & Evaluation

Training Random Forest, LSTM and 1D-CNN models on collected dataset. Evaluating prediction accuracy, false positive rate and inference latency on Raspberry Pi 4. Flask API integration ongoing.

PHASE 07 · PLANNED

Predictive RUL Estimation & Alerts

Remaining Useful Life (RUL) regression model. Automated Grafana alert notifications (SMS/email) on threshold breach. Continuous model retraining pipeline with new operational data.
